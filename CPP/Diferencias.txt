6. se castea diferente: 

En C usas el cast estilo (int)variable. C++ tiene sus propios operadores de casting que son más fáciles de buscar en el código y más seguros (aunque el estilo C sigue funcionando, se desaconseja).

C: int x = (int)3.14;

C++: int x = static_cast<int>(3.14);


Tipo de Cast,Qué hace,Nivel de Riesgo
static_cast,"Conversiones normales (int a float, puntero a clase base). Verifica en compilación que tenga sentido.",Bajo (Seguro)
const_cast,Únicamente para quitar o poner el atributo const.,Medio
reinterpret_cast,Trata los bits de un tipo como si fueran otro (como castear un long a un puntero). Es el más parecido al estilo C.,Alto (Peligroso)
dynamic_cast,La joya de la corona. Verifica en tiempo de ejecución si un puntero a una clase padre realmente apunta a una clase hija específica.,Muy Seguro

8.  CLASES ******************************:

A. Encapsulamiento (public vs private)
En C, cualquier parte del programa puede modificar un struct. En C++, puedes proteger los datos.

private: Solo las funciones dentro de la clase pueden ver esto. Evita que alguien haga miAuto.velocidad = -500; por error.

public: Es la interfaz que expones al usuario.

B. El Constructor y Destructor
Olvídate de funciones como crear_usuario() y destruir_usuario().

Constructor (NombreClase()): Se ejecuta automáticamente cuando creas el objeto. Sirve para inicializar memoria o variables.

Destructor (~NombreClase()): Se ejecuta automáticamente cuando el objeto sale de memoria (cuando termina el main o un if). Es ideal para hacer free() o cerrar archivos sin que se te olvide.

C. Herencia
Puedes crear una clase base y que otras "hereden" sus poderes. Por ejemplo, una clase Vehiculo y que Coche y Moto hereden de ella para no repetir código.

9. ----------------------------------MEMORIAAAAAAAAAAAAAAAAAAA ---------------------------------------

Aquí está la regla de oro:

Sin new: Vive en el Stack (Memoria automática). VARIABLE COMO TAL
Con new: Vive en el Heap (Memoria dinámica). PUNTERO AL HEAP. 


En c: 
#include <stdio.h>
#include <stdlib.h>

// Definimos la "clase" como un struct
struct Archivo {
    FILE* fp;
};

// El "Constructor" manual
struct Archivo abrir_archivo(const char* nombre) {
    struct Archivo a;
    a.fp = fopen(nombre, "w");
    if (a.fp == NULL) {
        perror("Error al abrir");
    }
    printf("Archivo abierto manualmente.\n");
    return a;
}

// El "Destructor" manual
void cerrar_archivo(struct Archivo* a) {
    if (a->fp != NULL) {
        fclose(a->fp);
        printf("Archivo cerrado manualmente.\n");
    }
}

void funcion() {
    struct Archivo miArchivo = abrir_archivo("datos.txt");
    
    // ... usamos el archivo ...
    fprintf(miArchivo.fp, "Hola desde C");

    // ¡OJO! En C, si te olvidas de esta línea, el archivo se queda abierto
    // hasta que el sistema operativo mate el proceso.
    cerrar_archivo(&miArchivo); 
}


en cpp: 
class Archivo {
private:
    FILE* fp;
public:
    // Constructor
    Archivo(const char* nombre) {
        fp = fopen(nombre, "w");
    }

    // Destructor
    ~Archivo() {
        if (fp) fclose(fp);
    }
};

void funcion() {
    Archivo miArchivo("datos.txt"); // Vive en el STACK
    // ... usas el archivo ...
} // <-- AQUÍ miArchivo muere y se ejecuta ~Archivo() automáticamente. SOLO SI ESTA EN STACK (no uso new).

Si fuera con new (o heap): 

void funcion() {
    Archivo* miArchivo = new Archivo("datos.txt"); // Puntero en stack, objeto en HEAP //CON NEW SE USA UN PUNTERO, crea el malloc y tambien llama al constructor En C: struct Archivo* a = malloc(sizeof(struct Archivo));.
    
    // ... usas el archivo ...
    
    delete miArchivo; // <-- AQUÍ se ejecuta ~Archivo() y se libera la memoria.
}

----------delete ptr: 
Una precisión técnica sobre el delete
Cuando haces delete ptr;, ocurren dos cosas en orden:

Se ejecuta el Destructor del objeto al que apunta ptr (para que el objeto limpie sus asuntos internos).

Se libera la Memoria en el heap (lo que en C haría free).

Si en C++ usaras free(ptr) con un objeto, la memoria se liberaría, pero el destructor nunca se ejecutaría, y si ese objeto tenía archivos abiertos o más memoria interna, habrías creado un desastre (un "leak" de recursos).

IMPORTANTE: Si usas new, usas delete. Si usas new[] (para arrays), usas delete[].

2. Los Scopes Internos (Loops, Ifs, Bloques sueltos)
Aquí es donde C++ brilla frente a C. Los loops o bloques { } no crean un "Stack Frame" nuevo (siguen en el mismo bloque de la función), pero sí definen un Scope.

Cualquier objeto creado dentro de un { } tiene "vida" solo dentro de esas llaves.

        // ... usar temp ...
    } // <-- AQUÍ MUERE el objeto 'temp' en CADA iteración.

}

11. Estructuras -------------------------------------------------------------------------------
C++ no decide "esto es muy grande, lo mando al heap". Lo que pasa es que std::vector (y otras estructuras como std::string) están programadas internamente para pedir memoria al heap en su Constructor.

El "Esqueleto" vs. El "Contenido"
Cuando haces std::vector<int> v(1000000);, el objeto v se divide en dos partes:

En el Stack (El control): El objeto v en sí es muy pequeño (usualmente 24 o 32 bytes). Solo guarda tres cosas: un puntero al inicio de los datos, el tamaño actual y la capacidad.

En el Heap (La carga): El constructor del vector ve que le pediste un millón de enteros, así que internamente hace un new int[1000000]. Ese millón de enteros vive en el Heap.


void funcion() {
    int arr[1000000]; // ¡ESTO ESTÁ EN EL STACK!
    // Si el stack de tu sistema es de 1MB o 2MB, 
    // lo más probable es que sufras un "Stack Overflow" y el programa explote.
}

void funcion() {
    std::vector<int> v(1000000); // El "control" está en el stack, los datos en el heap.
    // El heap tiene mucha más memoria disponible (toda tu RAM),
    // así que esto es seguro.
}


12. Punteros ------------------------------------------------------------------------------
El valor de "Nada": NULL vs nullptr
En C, NULL es simplemente un #define de 0. Esto causa problemas porque el compilador no sabe si te refieres al número cero o a una dirección de memoria.

En C: int *p = NULL; (Es un entero 0 convertido a puntero).

En C++: int *p = nullptr; (nullptr tiene su propio tipo de dato: nullptr_t). Nunca se confunde con un entero.




**** Smart Pointer *******
4. Punteros Inteligentes (Smart Pointers)
Esta es la diferencia más grande. En C++ moderno (C++11 en adelante), casi nunca deberías usar new o delete manualmente. Usamos objetos que manejan el puntero por nosotros.

A. std::unique_ptr (El dueño único)
Se asegura de que solo exista UN puntero a esa memoria. Cuando el puntero sale del stack (llave de cierre }), hace el delete automáticamente.

C++
#include <memory>

void funcion() {
    std::unique_ptr<int> p = std::make_unique<int>(100);
    // No hace falta 'delete', se limpia solo al terminar la función.
}


**** shared pointer *****

B. std::shared_ptr (Dueños compartidos)
Varios punteros apuntan al mismo objeto. El objeto solo se borra cuando el último de los punteros muere (lleva un contador interno).


**** this **** 
El Puntero this
En C++, todas las funciones dentro de una clase tienen un puntero oculto llamado this. Es la dirección del objeto actual.

C++
class Persona {
    int edad;
public:
    void setEdad(int edad) {
        this->edad = edad; // 'this->edad' es la variable de la clase, 'edad' es el parámetro.
    }
};


**** Aritmética de Punteros vs. Iteradores ****
En C hacías p++ para recorrer un array. En C++ usamos Iteradores. Son como punteros con esteroides que funcionan igual para arrays, listas enlazadas o mapas.

C++
std::vector<int> v = {1, 2, 3};
for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it; // Se usa igual que un puntero
}


****** Referencias ******
        1. En C (Usando Punteros)
        Tienes que pasar la dirección y usar asteriscos dentro de la función. Es propenso a errores (¿qué pasa si el puntero es NULL?).
        
        C
        void incrementar(int *n) {
            if (n != NULL) { // Validación manual obligatoria
                (*n)++;      // Sintaxis ruidosa
            }
        }
        
        int main() {
            int x = 5;
            incrementar(&x); // Tienes que acordarte del '&'
            return 0;
        }
        2. En C++ (Usando Referencias)
        La función recibe la variable directamente, pero el símbolo & en el parámetro le dice al compilador: "No hagas una copia, usa la variable original".
        
        C++
        void incrementar(int &n) { // El '&' aquí indica referencia
            n++;                   // ¡Se usa como una variable normal!
        }
        
        int main() {
            int x = 5;
            incrementar(x);        // ¡No necesitas el '&' aquí! Es transparente.
            return 0;
        }

14. Estructuras de datos y Templates: 
            
            1. std::vector (El Array Dinámico Inteligente)
            Es la estructura más usada en C++. Es un array que crece solo. Olvídate de realloc.
            
            En C: Tienes que llevar la cuenta del size, la capacity y hacer realloc manualmente.
            
            En C++: v.push_back(valor); y el vector se encarga de todo.
            
            2. std::string (Cadenas de texto de verdad)
            En C, un string es un char* amargado. En C++, std::string es una estructura de datos completa.
            
            Puedes sumarlos: str1 + str2.
            
            Puedes compararlos: if (str1 == str2).
            
            Se encargan de su propia memoria.
            
            3. std::map y std::unordered_map (Diccionarios / Hash Tables)
            Esto no existe en C estándar. Te permite guardar pares de Clave-Valor.
            
            C++
            #include <unordered_map>
            std::unordered_map<string, int> edades;
            edades["Leo"] = 35; // Inserta o busca en tiempo récord (O(1))
            4. std::list (Lista Doblemente Enlazada)
            Ya no tienes que crear el struct Nodo { struct Nodo* next; ... }. C++ te da la lista lista para usar con métodos como .sort(), .reverse() y .merge().
            
            5. std::stack, std::queue y std::priority_queue
            Estructuras de tipo LIFO (Pilas), FIFO (Colas) y Colas de prioridad (donde el elemento más grande siempre sale primero).
            
            
            La gran ventaja: Los "Templates" (Plantillas)
            En C, si hacias una lista de int, no servía para float. Tenías que usar void* y perder la seguridad de tipos.
            En C++, las estructuras usan Templates, lo que permite que funcionen con cualquier tipo de dato manteniendo la velocidad del código nativo.
            
            C++
            std::vector<int> miListaDeEnteros;
            std::vector<Persona> miListaDeObjetos; // ¡Cualquier cosa entra aquí!
