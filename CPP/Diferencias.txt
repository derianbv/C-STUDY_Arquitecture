6. se castea diferente: 

En C usas el cast estilo (int)variable. C++ tiene sus propios operadores de casting que son más fáciles de buscar en el código y más seguros (aunque el estilo C sigue funcionando, se desaconseja).

C: int x = (int)3.14;

C++: int x = static_cast<int>(3.14);

8.  CLASES ******************************:

A. Encapsulamiento (public vs private)
En C, cualquier parte del programa puede modificar un struct. En C++, puedes proteger los datos.

private: Solo las funciones dentro de la clase pueden ver esto. Evita que alguien haga miAuto.velocidad = -500; por error.

public: Es la interfaz que expones al usuario.

B. El Constructor y Destructor
Olvídate de funciones como crear_usuario() y destruir_usuario().

Constructor (NombreClase()): Se ejecuta automáticamente cuando creas el objeto. Sirve para inicializar memoria o variables.

Destructor (~NombreClase()): Se ejecuta automáticamente cuando el objeto sale de memoria (cuando termina el main o un if). Es ideal para hacer free() o cerrar archivos sin que se te olvide.

C. Herencia
Puedes crear una clase base y que otras "hereden" sus poderes. Por ejemplo, una clase Vehiculo y que Coche y Moto hereden de ella para no repetir código.

Aquí está la regla de oro:

Sin new: Vive en el Stack (Memoria automática). VARIABLE COMO TAL
Con new: Vive en el Heap (Memoria dinámica). PUNTERO AL HEAP. 


En c: 
#include <stdio.h>
#include <stdlib.h>

// Definimos la "clase" como un struct
struct Archivo {
    FILE* fp;
};

// El "Constructor" manual
struct Archivo abrir_archivo(const char* nombre) {
    struct Archivo a;
    a.fp = fopen(nombre, "w");
    if (a.fp == NULL) {
        perror("Error al abrir");
    }
    printf("Archivo abierto manualmente.\n");
    return a;
}

// El "Destructor" manual
void cerrar_archivo(struct Archivo* a) {
    if (a->fp != NULL) {
        fclose(a->fp);
        printf("Archivo cerrado manualmente.\n");
    }
}

void funcion() {
    struct Archivo miArchivo = abrir_archivo("datos.txt");
    
    // ... usamos el archivo ...
    fprintf(miArchivo.fp, "Hola desde C");

    // ¡OJO! En C, si te olvidas de esta línea, el archivo se queda abierto
    // hasta que el sistema operativo mate el proceso.
    cerrar_archivo(&miArchivo); 
}


en cpp: 
class Archivo {
private:
    FILE* fp;
public:
    // Constructor
    Archivo(const char* nombre) {
        fp = fopen(nombre, "w");
    }

    // Destructor
    ~Archivo() {
        if (fp) fclose(fp);
    }
};

void funcion() {
    Archivo miArchivo("datos.txt"); // Vive en el STACK
    // ... usas el archivo ...
} // <-- AQUÍ miArchivo muere y se ejecuta ~Archivo() automáticamente. SOLO SI ESTA EN STACK (no uso new).

Si fuera con new (o heap): 

void funcion() {
    Archivo* miArchivo = new Archivo("datos.txt"); // Puntero en stack, objeto en HEAP //CON NEW SE USA UN PUNTERO, crea el malloc y tambien llama al constructor En C: struct Archivo* a = malloc(sizeof(struct Archivo));.
    
    // ... usas el archivo ...
    
    delete miArchivo; // <-- AQUÍ se ejecuta ~Archivo() y se libera la memoria.
}
